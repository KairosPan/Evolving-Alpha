## 2. 核心不变量与约定(动手前必须知道)

这些贯穿全代码,新增任何东西都要遵守。多次终审就是在守这些。

1. **未来函数防火墙(头号铁律)**。任何"决策时刻"的代码只能用 ≤t 的信息。
   - 取数一律经 `GuardedSource`(`AsOfGuard.check(day)`,请求未来日 → `LookaheadError`)。
   - 评测打分用**延迟打分**(`WalkForwardEval`):t 决策、游标自然推进到 t+horizon 才打分。
   - "已实现未来"oracle/teacher 的未来标签**只进训练/打分,永不进决策推理路径**。
   - 防御还来自**结构**:`MarketState`/`StockSnapshot`/`CandidateUniverse` 是 frozen、无 source 句柄的快照——拿到它的策略**够不到**任意日期。
2. **Harness `H=(p,G,K,M)`**:`p`=doctrine(`Doctrine`)· `K`=技能(`SkillRegistry`)· `M`=记忆(`MemoryStore`)· 情绪周期状态机(`StateMachine`)。G(子 Agent)= Phase-1 的 LLM 模块,目前只有主 Agent。**结构性编辑(改打法)只能经 `MetaTools` 的 9 个 meta-tool**(write/patch/retire→dormant/revive/promote_skill、process/update/demote_memory、rewrite_doctrine),且进 `EditLog` 审计。
   - **观测 vs 编辑边界(1b-1 确立)**:`SkillStats`(n/wins/losses/nukes/ewma/expectancy)是**观测**——`refine.apply_credit` 按已实现 oracle 结果**直写**,不算结构性编辑、**不入 EditLog**;`EditLog` 只记 Refiner 的结构性 Δ。stats 仍被快照捕获,rollback 覆盖。
3. **immutable-core 写保护**:`DoctrineEntry.immutable=True` 的纪律红线,`__setattr__` 守卫 + `Doctrine.rewrite/remove` 双重拦截,Refiner 永远改不动。
4. **已实现未来 = 交易版 Dijkstra**(`eval/oracle`):pool-membership 类别(次日仍涨停=continued / 跌停炸板=nuked / 掉出=faded)。这是量化"决策好不好"的客观尺。**策略(含 LLM Agent)都是 `DecisionPolicy`,统一进 `WalkForwardEval` 量化。**
5. **代码约定**(照抄现有风格):
   - PIT 快照模型 = **frozen pydantic**(`ConfigDict(frozen=True)`);种子载入模型 = **`extra="forbid"`**(catch 拼写错);可被 patch 的 = **`validate_assignment=True`**。
   - 容器类(SkillRegistry/MemoryStore/CandidateUniverse/EditLog)= **`__bool__ = True`**(杀 falsy-empty 陷阱,曾两次中招)。
   - **缺失值诚实 `None`,不臆造 0/""**;`pd.isna(v)` 判缺(含 NaT)。
   - **regime 归一**:`regime.split_regimes`/`parse_regime_field` 把不统一的相位词归一到 7 个 canonical 相位(`混沌冰点/修复启动/情绪回暖/题材启动/主升/震荡补涨/退潮`)+ 生态标签。
   - LLM 输出**一律不可信**:`agent/parse` 过滤幻觉 code(只留候选池内的)、钳 confidence、malformed→空仓兜底。
6. **离线可测**:测试用 `FakeSource`(行情)/ `MockLLMClient`(LLM),永不触网。真实 akshare/DeepSeek 只在 `scripts/smoke_*.py`(手动)。

## 3. 开发工作流(本项目一直这么干,建议沿用)

每个阶段一个完整循环,产出可工作、可测、已评审的软件:

1. **写计划**(`superpowers:writing-plans`):落到 `docs/superpowers/plans/YYYY-MM-DD-<topic>.md`,bite-sized TDD 任务、精确路径/完整代码/命令、自检(spec 覆盖/无 placeholder/类型一致)。**范围切到"一个可独立测试的单元"**,大了就拆(如 Phase-0b 拆 1/2/3)。
2. **subagent-driven 执行**(`superpowers:subagent-driven-development`):
   - 把任务**按层 bundle**(3-4 个),每 bundle 派一个 implementer 子 Agent(TDD、per-task commit)。
   - 每 bundle **两段评审**:① spec 合规(对照计划逐行)② 代码质量(对抗性找真问题)。评审发现 → 同一 implementer 修 → 复核。
   - 全部 bundle 完 → **opus 终审**(集成级:不变量、防火墙、真实数据鲁棒性)。
3. **收尾**(`superpowers:finishing-a-development-branch`):feature 分支 → 验证 merged main 全绿 → 合并(FF)→ 删分支。每阶段更新 `PROJECT_STATE.md` + memory。
- **git**:`main` 受保护,功能在 `phase-xx` 分支做;commit message 中文、按 feat/fix/refactor/test/docs/chore 分类。
- 评审拦下的真问题样本(说明评审值):falsy-empty-log 静默丢弃、`__setattr__` 绕过 immutable、NaT→"None" 字符串、半改污染、幻觉 code、未来函数侧信道泄漏。**这些都不是走过场。**

## 5. 已知债务清单(汇总自各阶段终审,按优先级)

**P0(实盘/真数据前必须解决)**
- **真实列名核对**:跌停池封单列是 `封单资金`≠`封板资金`;zt 池可能 `最后封板时间`≠`首次封板时间`。跑 smoke 核对并修 `data/source.py` 的 `_RENAME`(离线测试测不到)。
- **幸存者偏差 / PIT 成分**:akshare 概念成分是当前成分;退市/ST/改名/当时可交易池未处理。**当前防火墙只防日期 lookahead,不防 survivorship**。需 point-in-time 数据治理 + 负例(看着像但失败了的)。
- **DeepSeekClient 无 retry/backoff**:多日实盘 eval 前必加(否则单次 API 错误中断全程);或在 `LLMAgentPolicy.decide` 包 try/except → 空仓。

**P1(Phase-1b 落地时处理)**
- 提示**全量注入**(55 技能+全 doctrine,~24KB/次)未按 `regime_read` 选择性注入;`win` 类记忆 + 技能 `stats/oracle_gap` 未注入(Refiner 产出后补,让 agent 看到"这技能在亏")。
- `EditRecord.payload` 多数 op 只记新值无 before-image → 当前回滚靠整版快照(够用);若要 delta-replay 回滚需补 before-image。
- `MetaTools.h/log` 公开可绕审计(soft 边界);PITStore 已实现但**未接入取数路径**(应作 read-through 插进 GuardedSource 内层,实现"边跑边快照")。
- 评测:`HighestBoard` 平局选全部 → n_candidates 偏大,比 agent 时需归一;expectancy 是**毛收益**(无成本)。

**P2(增强 / 提升进化质量)**
- `_extract_json` 对 thinking-blob/多 JSON 对象前缀目前安全兜底但不解析(改贪婪 json 扫描)。
- 3 条种子非 canonical token(`次新/启动`、`连板/主升后期/情绪极值` 等)只留在 `regime_raw`,`for_regime` 查不到;Refiner/种子curation 修。

**Phase-1b-1 推迟项(随能力解锁,非阻塞)**
